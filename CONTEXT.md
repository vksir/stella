# Stella

Stella 是一个 AI 智能体：通过多平台接入的对话伙伴，同时也是带工具的干活 Agent。本文件是项目的领域术语表。

## Language

**Stella**:
本项目的智能体本体。既提供对话能力，也持有可执行工作的工具集。

**平台 (Platform)**:
消息进出 Stella 的接入点。当前有两个：QQ（经 napcat，OneBot v11 协议）与 HTTP API（自动生成文档）。
_Avoid_: 渠道、端

**用户 (User)**:
Stella 识别的对话者实体，跨平台唯一。记忆与权限都挂在用户上；经平台身份解析识别，首次接触未平台身份时自动建档。角色分 admin 与 guest。
_Avoid_: 账号

**平台身份 (Platform Identity)**:
用户在某平台上的标识凭据（如 QQ 号、API token）。一个用户可拥有多个平台身份；多个平台身份关联到同一用户后，该用户在各平台被识别为同一人。

**主人 (Owner)**:
配置文件中登记的唯一高权限用户（role=admin），通过其平台身份（QQ 号、API token）识别。可使用全部工具。
_Avoid_: 管理员、root

**白名单工具 (Whitelisted Tool)**:
配置指定的一组按构造就安全的工具——副作用（若有）严格封闭在调用者自身的数据域内（如记忆工具只能写当前说话人自己的记忆）——对所有平台的全部用户开放。不在白名单中的工具一律仅主人可用。
_Avoid_: 公开工具、免费工具

**会话 (Session)**:
Stella 的一条持久对话上下文，短期记忆即其历史。QQ 私聊按用户划分（每用户一条）；QQ 群聊按群划分（每群一条，全群成员共享）；API 由调用方以会话 id 指定，缺省新建。
_Avoid_: 对话、线程

**短期记忆 (Short-term Memory)**:
即会话历史本身，随会话存续；Stella 实际可见的历史受双重淘汰约束——token 量超上限（可配置，默认 10K）或年龄超 3 天的最旧消息自动丢弃。
_Avoid_: 上下文缓存

**触发 (Trigger)**:
决定 Stella 是否回应一条消息的规则：私聊逐条必回；群聊仅被 @ 时回应（@全体成员不算被 @）。

**倾听 (Listening)**:
群聊中未触发回应的消息仍写入会话历史，使 Stella 被 @ 时握有上文。

**长期记忆 (Long-term Memory)**:
跨会话存续的用户关键信息（如偏好），与产生它的用户关联。存取完全由 Stella 通过记忆工具自主完成，系统不做自动提取或注入；工具作用域限于当前说话人，不能跨用户读写。v1 不设挂在群上的共享记忆。
_Avoid_: 用户画像、群记忆
