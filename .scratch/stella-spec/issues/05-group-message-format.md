# 群聊消息编排与 @ 回复格式

Type: prototype
Status: closed (2026-07-19)
Blocked by: 01, 03

## Question

群聊是多用户共享一条会话，需要定一套消息编排格式，并做出粗样给用户感受后拍板：

- 注入侧：群消息写入会话历史时如何标注说话人（昵称/QQ 号/时间）？与私聊消息如何区分？"当前谁在 @ Stella 提问"如何在当轮请求里标明？
- 输出侧：Stella 的回复如何路由回 QQ 并 @ 回复对象（OneBot at 段的构造）？她能否主动 @ 别人或引用消息？
- 该格式对 100K 窗口的占用代价。
- 由研究票新浮出的子问题：@全体成员（at 段 `qq: "all"`）算不算触发 Stella？（触发策略的细化，随本票一并拍板。）

产出：注入提示词样例 + 输出解析样例的粗原型，供用户拍板。依赖 [OneBot v11 / napcat 协议事实] 与 [PI SDK 深度行为事实] 中的每轮注入挂载点。

## Resolution

原型 `.scratch/stella-spec/prototype-05/`（`bun run proto:group-format`）演示后全部拍板。原型已作为一手来源固化在一次性分支 `proto/05-group-format`（commit f9b7a05），`format.ts` 纯函数可平移进实现：

1. **注入侧**：群消息入库标注 `[#消息id 群名片(QQ号) HH:MM] 内容`（群名片优先于昵称；@Stella → 行内 `@Stella`，@别人 → `@昵称(QQ号)`，@全体 → `@全体成员`；图片/表情等 → `[图片]` 占位；引用 → `[引用#id]`）。触发注记（`【本轮触发】…自动 @ 当前说话人…`）走 `before_agent_start` 当轮注入、不入库、当轮自动还原。私聊不标注说话人（会话按用户划分，身份写系统提示词）。窗口代价实测 ≈7 tok/条，占 10K 上限比例可忽略。
2. **输出侧**：群聊回复自动在开头 at 当前说话人；模型行首 `[reply:#消息id]` → reply 段（主动引用）；正文 `@QQ号`（5+ 位数字）→ at 段（主动提及）。标记解析失败退化为纯文本发送；引用过期 message_id（LRU 约 5000 条）报错时平台层降级为不带引用重发。
3. **@全体成员不触发**：`at` 段 `qq:"all"` 仅倾听入库；触发 = `at` 段 `qq === String(self_id)`。理由：@全体是广播非专门对 Stella 说话；避免批量唤起刷屏费 token。
