# v1 工具集与白名单盘点

Type: grilling
Status: closed (2026-07-19)

## Question

权限模型已定（白名单全员可用、非白名单仅主人），本票盘点 v1 的具体工具集：

- 主人全量工具清单：SDK 内置 read/bash/edit/write 全开吗？要不要 grep/find/ls？
- 群聊/全员白名单放哪几个工具？候选如网页搜索——是否引入？引入则需定义自定义搜索工具（接哪家搜索 API？）。
- 除记忆工具外还需要哪些自定义工具？
- 记忆工具本身对全员开放还是仅主人？（存自己的记忆应当是全员能力——与"作用域 = 当前说话人"自洽，但需确认。）

## Resolution

1. **主人工具 = SDK 内置 7 件全开**（read/bash/edit/write/grep/find/ls）。理由：bash 在能力上等价一切，收窄只读工具只是逼模型绕路；专用工具更省 token。
2. **全员白名单 = 仅记忆工具**。内置 read/grep/find/ls 虽只读，但暴露主人整机文件系统（含配置里的密钥），不满足"按构造就安全"，不进白名单；不引入网页搜索（外部依赖+计费+滥用面，演进时单独开研究票选型）。
3. **v1 自定义工具 = 仅记忆工具**，无其他（主动发话不在 v1 使命；会话/记忆管理属 API 层不复制到工具面）。
4. **记忆工具全员开放**，作用域 = 当前说话人（与 CONTEXT.md 长期记忆定义自洽）；群友刷条目只污染其自身记忆区，按用户条目数上限的缓释留待票 07。
